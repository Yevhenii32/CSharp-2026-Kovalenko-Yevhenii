# Менеджер товарів (Product Manager)

Цей проєкт є виконанням Лабораторної роботи 1 з курсу С#.
Тема: "Основи С# та ООП та проєктування класів. Основи роботи з колекціями".

## Опис проєкту
Застосунок призначений для обліку складів і товарів, що належать до цих складів. Проєкт побудований з використанням принципів об'єктно-орієнтованого програмування, багатошарової архітектури та принципу Single Responsibility.

## Структура рішення (Solution)
Рішення розділене на 4 окремі проєкти:

1. **ProductManager.Models** (Бібліотека класів)
   - Містить базові моделі зберігання (`Warehouse`, `Product`).
   - Класи не містять обчислюваних полів та прямих посилань один на одного (зв'язок реалізовано через `WarehouseId`).
   - Містить перечислення (`Location`, `ProductCategory`).

2. **ProductManager.ViewModels** (Бібліотека класів)
   - Містить моделі для відображення (`WarehouseViewModel`, `ProductViewModel`).
   - Відповідає за обчислювані поля (наприклад, загальна вартість товару та загальна вартість товарів на складі).

3. **ProductManager.Services** (Бібліотека класів)
   - `TempStorage`: статичний `internal` клас, що імітує базу даних (заповнюється 3 складами та 12 товарами при запуску).
   - `StorageService`: публічний сервіс, що надає методи для безпечної взаємодії зі сховищем.

4. **ProductManager.ConsoleApp** (Консольний застосунок)
   - Головна точка входу.
   - Реалізує текстовий інтерфейс користувача для перегляду списку складів, вибору конкретного складу та детального перегляду його товарів.

## Особливості реалізації
- Використано тип `decimal` для фінансових обчислень (ціна товарів).
- В консолі налаштовано кодування UTF-8 для коректного відображення української мови.
- Реалізовано безпечний доступ до даних: колекції з `TempStorage` не передаються напряму, а лише читаються через `StorageService`.
